# 代码预审查检查清单

## 安全与稳定性检查

### 1. 内存安全
- [ ] **大文件处理**：是否使用流式读取（BufReader）而非 `read_to_string`？
- [ ] **无界数据结构**：Vec/HashMap 是否有明确的大小限制？
- [ ] **递归深度**：递归函数是否有深度限制？

### 2. 数据库操作
- [ ] **迁移幂等性**：`ALTER TABLE` 前是否检查列/表已存在？
- [ ] **冲突处理**：`ON CONFLICT` 是否清除软删除标记？
- [ ] **SQL 注入**：是否使用参数化查询？
- [ ] **事务**：多步操作是否包裹在事务中？

### 3. 输入验证
- [ ] **文件上传**：是否有大小限制（DefaultBodyLimit）？
- [ ] **路径遍历**：是否验证路径不包含 `../`？
- [ ] **空值处理**：是否处理 `None`/`null` 情况？
- [ ] **超大输入**：字符串/数组是否有长度限制？

### 4. 并发与资源
- [ ] **资源释放**：文件句柄、连接是否正确关闭？
- [ ] **竞争条件**：是否有并发修改共享数据？
- [ ] **超时**：网络/IO 操作是否有超时设置？

### 5. 错误处理
- [ ] **错误传播**：是否滥用 `unwrap`/`expect`？
- [ ] **用户提示**：错误信息是否暴露敏感信息？
- [ ] **日志**：关键操作是否有日志记录？

## 代码质量检查

### 6. API 设计
- [ ] **命名误导**：方法名是否准确反映行为？
- [ ] **副作用**：函数是否有未声明的副作用？
- [ ] **返回值**：错误码/结果是否一致？

### 7. 测试
- [ ] **边界测试**：是否测试空值、极大值、异常输入？
- [ ] **重复测试**：多次执行是否得到一致结果？
