# Decision Log — 桌宠内嵌视图全屏交互优化

## 2026-06-03

### D-1: 全屏模式范围
- **决策**: 仅限主窗口内嵌宠物视图（DivaPetView in NormalMode），不涉及桌面弹出窗口（DesktopPetOverlay）
- **来源**: 用户确认（Brain dump Q2）

### D-2: 5秒自动消失范围
- **决策**: 仅限宠物页面的 overlay 侧边栏，不推广到其他页面
- **来源**: 用户确认（Brain dump Q3）

### D-3: 参考实现模式
- **决策**: 采用 `NormalMode.vue.before-refactor` 的 overlay 侧边栏模式（汉堡按钮 → 浮层 + 遮罩 → 点击关闭）
- **来源**: 用户指示（"参考一下原来怎么做的"）

### D-4: 汉堡按钮位置
- **决策**: 复用 DivaPetView 现有左上角 Menu 按钮（`pet-edge-button absolute top-4 left-4`），另在 NormalMode 新增
- **来源**: 用户确认（Fast path Q1）

### D-5: 页面切换时侧边栏行为
- **决策**: 离开宠物页面时侧边栏恢复显示
- **来源**: 用户确认（Fast path Q2）

### D-6: 迷你状态栏形态
- **决策**: 浮动 overlay（pet-glass 半透明样式），非固定顶栏
- **来源**: 用户确认（Fast path Q3）

### D-7: 工作模式
- **决策**: Fast path — 批量收集缺口后直接起草 PRD
- **来源**: 用户选择

### D-8: PRD 定位
- **决策**: Internal Tool 级别，~3 页，5 个 FRs
- **来源**: Stakes 校准（默认假设）
