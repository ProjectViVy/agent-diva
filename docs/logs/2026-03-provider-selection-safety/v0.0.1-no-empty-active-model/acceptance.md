# Acceptance

1. Open `Settings -> Providers` and select a provider with an active model.
2. Click the currently active model row again.
3. Confirm the model is removed from shortcuts only, and the active provider/model config is not cleared.
4. Delete a currently active custom model when another model is available.
5. Confirm the UI switches to a fallback model and the gateway can still start.
6. Attempt to delete the only active custom model for a provider.
7. Confirm the deletion is refused instead of saving an empty active model.
