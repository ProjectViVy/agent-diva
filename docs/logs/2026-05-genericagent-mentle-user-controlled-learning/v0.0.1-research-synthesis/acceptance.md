# Acceptance

## 验收目标
确认“GenericAgent 优雅接入 + 用户可控学习”方案满足当前阶段产品约束：
- 不引入 laputa 全栈复杂度
- 不牺牲长期演进能力
- 学习沉淀完全用户可控

## 验收清单

- [x] 方案明确区分三层职责:
  - Compass: `MEMORY.md`
  - Deep Memory: mentle
  - Navigation: 索引文件
- [x] 学习链路闭环完整:
  - 候选提取 -> Inbox -> 节律提问 -> 用户决策 -> 写入/丢弃 -> 召回
- [x] 决策选项覆盖:
  - SOP / Skill / Fact / History / Discard
- [x] 治理机制完整:
  - 审计日志、撤销、重分类、静默期、限频
- [x] 召回路径可解释:
  - 索引优先 -> mentle 定向检索 -> Compass 回退
- [x] 输出为调研文档，不含代码改动

## 产品确认问题（供下一阶段）
1. 每日提问默认时刻与上限是否固定，是否需要 per-user 配置？
2. `SOP` 与 `Skill` 的最小区分标准是否采用统一模板？
3. 索引文件最终是否要迁移为结构化存储（如 SQLite 表）？
4. 首批上线是否仅开启 `Fact/History` 两类以降低分类复杂度？

## 验收结论
- 本版本满足“可研阶段”验收，建议进入接口草案与实现排期阶段。
