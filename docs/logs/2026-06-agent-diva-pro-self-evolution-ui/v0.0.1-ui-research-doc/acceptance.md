# Acceptance

## Expected Outcome

Another engineer should be able to read the new document and answer:

- what self-evolution UI surfaces must be built first;
- where those surfaces belong in the existing GUI;
- which new card types are required;
- which new manager/API routes are implied;
- which storage boundaries must not be crossed by the GUI;
- what the recommended implementation order is for `agent-diva-pro`.

## User-facing Acceptance

This iteration is accepted when the repository contains a dedicated, standalone self-evolution UI research document and matching iteration log records, without mixing those changes into unrelated in-progress research files.
