# 10 - UI 设计文档

> Diva Pet 模块的界面设计与交互规范

---

## 1. 设计理念

Diva Pet 的目标是在 Agent Diva 原有的 "工具型" 界面基础上，增加一个 "陪伴型" 交互层。设计原则：

1. **不干扰现有工作流**：桌宠作为可选页面/独立窗口，不侵占 Chat 对话空间
2. **渐进式交互**：从视觉（Live2D）→ 听觉（TTS）→ 语音（ASR），逐步增加沉浸感
3. **简洁克制**：避免功能堆砌，聚焦核心体验

## 3. 页面状态设计

### 3.1 状态机

```
                 ┌──────────┐
          ┌─────▶│  空闲    │◀────────┐
          │      │ (Idle)   │         │
          │      └────┬─────┘         │
          │           │               │
    用户不活动        │ 收到消息       │ 回复完成
          │      ┌────▼─────┐         │
          │      │  回复中   │─────────┘
          │      │ (Speaking)│
          │      └──────────┘
          │
          ▼
    ┌──────────┐
    │  休眠    │  ← 长时间无交互 / 窗口不可见
    │ (Sleep)  │
    └──────────┘
```

### 3.2 各状态视觉效果

| 状态 | Live2D 表情 | 动作 | 气泡 |
|------|------------|------|------|
| **空闲** | 微笑 + 呼吸 | Idle 动作组（随机） | 隐藏 / 随机问候 |
| **思考中** | 思考表情 | Attentive 动作 | "让我想想..." |
| **回复中** | 说话表情 | Speaking 动作 | 流式显示回复文本 |
| **工具执行** | 专注表情 | Working 动作 | "正在处理..." |
| **错误** | 困惑表情 | Shy/Embarrassed | 错误提示文字 |
| **休眠** | 闭眼 | Sleep 动作（如果有） | 隐藏 |
| **被拖拽** | 惊讶表情 | Dragging 动作 | "Hey!" |

## 4. 关键交互设计

### 4.1 点击交互

| 点击位置 | 行为 | Live2D 反馈 |
|----------|------|------------|
| **头部** | 唤起对话（发送问候） | 开心表情 + Head Tap 动作 |
| **身体** | 拖拽窗口 | Dragging 动作 |
| **气泡** | 展开/收起 | 无 |

### 4.2 语音交互流程

```
用户操作：
  1. 点击麦克风按钮 🎤
     → 按钮变为激活态 (红色脉冲动画)
     → Live2D 变为聆听表情
  2. 用户说话...
     → 气泡实时显示识别文本（浅色）
  3. 识别完成
     → 文本变为确认色
     → 自动发送到 Agent Loop
  4. Agent 回复
     → Live2D 切换为说话动作
     → 气泡显示回复（流式更新）
     → TTS 同步播报
  5. 播报完成
     → 麦克风自动恢复监听
```

## 6. 配色方案

### 6.1 主题色

由于 agent-diva 使用 `themeMode`（light/dark），Diva Pet 组件需适配两套主题。

```css
/* Light Mode */
:root {
  --pet-bg: rgba(255, 255, 255, 0.85);
  --pet-bubble-bg: rgba(255, 255, 255, 0.92);
  --pet-text: #1a1a2e;
  --pet-accent: #7c3aed;       /* violet-500 */
  --pet-mic-active: #ef4444;   /* red-500 */
  --pet-border: rgba(0, 0, 0, 0.08);
}

/* Dark Mode */
.dark {
  --pet-bg: rgba(30, 30, 46, 0.85);
  --pet-bubble-bg: rgba(40, 40, 60, 0.92);
  --pet-text: #e2e8f0;
  --pet-accent: #a78bfa;       /* violet-400 */
  --pet-mic-active: #f87171;   /* red-400 */
  --pet-border: rgba(255, 255, 255, 0.08);
}
```

### 6.2 动画

```css
/* 麦克风激活脉冲 */
@keyframes mic-pulse {
  0%, 100% { box-shadow: 0 0 0 0 rgba(239, 68, 68, 0.4); }
  50%      { box-shadow: 0 0 0 8px rgba(239, 68, 68, 0); }
}

/* 气泡弹出 */
@keyframes bubble-pop-in {
  0%   { opacity: 0; transform: translateY(8px) scale(0.95); }
  100% { opacity: 1; transform: translateY(0) scale(1); }
}

/* 流式文字光标 */
@keyframes cursor-blink {
  0%, 100% { opacity: 1; }
  50%      { opacity: 0; }
}
```

## 7. 响应式考量

虽然当前仅支持桌面端（Tauri），但组件应保持一定的尺寸弹性：

| 断点 | Live2D Canvas | 气泡宽度 | 控制栏 |
|------|---------------|----------|--------|
| 默认 (>800px) | 360×460 | 280px | inline |
| 紧凑 (600-800px) | 280×360 | 240px | inline |
| 极小 (<600px) | 200×260 | 200px | stacked |

> 注：极小尺寸主要用于独立小窗口模式（future feature）。

## 8. 无障碍（a11y）

| 元素 | ARIA 属性 | 键盘操作 |
|------|----------|----------|
| Live2D Canvas | `role="img"`, `aria-label="Diva 桌宠角色"` | — |
| 麦克风按钮 | `aria-label="语音输入"`, `aria-pressed` | `Enter` / `Space` |
| 气泡 | `aria-live="polite"` | — |
| 设置面板 | `role="dialog"`, `aria-modal="true"` | `Escape` 关闭 |

## 9. 独立窗口模式（Future）

未来可考虑将 Diva Pet 迁移到独立 Tauri 窗口：

```json
// tauri.conf.json
{
  "windows": [{
    "label": "pet",
    "title": "Diva",
    "width": 400,
    "height": 600,
    "decorations": false,
    "transparent": true,
    "alwaysOnTop": false,
    "skipTaskbar": true
  }]
}
```

此模式下，用户可在使用 Chat 的同时看到桌宠，类似 AniPet 的体验。
