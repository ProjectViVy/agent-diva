# v0.0.1 provider-selection-toggle-fix

## Summary
- 修复 `agent-diva-gui` 设置 -> 供应商页面中，左侧供应商无法再次点击取消选中的缺陷。
- 为当前主聊天已选模型补充“删除当前模型”入口，删除时同时清空当前 `provider/model` 并移除对应快捷模型。
- 为聊天页面顶部的模型切换下拉补充每项独立删除按钮，删除当前正在使用的模型时同步清空当前聊天配置。
- 调整模型卡点击语义为“切换并保存到快捷列表”，避免原有“同一点击既像切换又像删除”的冲突。
- 删除供应商页在线模型目录与 `LIVE/FALLBACK` 复杂交互，仅保留页面级手动刷新按钮。
- 修复 `agent-diva-manager` 配置更新链路对空 `provider/model` 的强制 fallback，允许真正取消激活当前模型。

## Impact
- 用户可以在供应商页返回未选择态，不再被迫保持某个供应商高亮。
- 用户可以显式删除当前主聊天模型，而不会被 `savedModels` 或自动同步逻辑立刻复活。
- 中英文 provider 文案补齐，新增删除当前模型的按钮文案与提示说明。
