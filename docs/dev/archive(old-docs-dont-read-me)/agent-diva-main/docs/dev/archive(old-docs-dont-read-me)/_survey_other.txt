 Summary: Defines the Plan Mode MVP architecture for agent-diva. Consolidates
   lessons from Codex, GenericAgent, and agent-diva into a native Rust runtime.
   Plan state lives in .agent-diva/plans/<plan-id>/ with state.json, events.jsonl,
   and typed domain models. Explicit non-goals: no GUI, no autonomous evolution,
   no parallel plan execution in MVP.

--- todolist-runtime-architecture.md ---
 First line: "# TodoList Runtime Architecture"
 Summary: Extends Plan Mode with a lightweight TodoList runtime — a rolling
   execution projection generated from a Plan. TodoList is NOT a second planning
   system; it is the short-horizon working queue. Kanban multi-worker dispatch
   and durable queues are intentionally deferred.
 Status: ARCHITECTURE DESIGN — complete design doc, not yet implemented.

 Summary: Turns the Plan Mode and TodoList architecture into a practical
   implementation route. Covers design options (file-backed JSON vs SQLite),
   phased delivery, and explicit deferral of Kanban/autonomous evolution.
   Target: user goal -> plan -> generated todolist -> serial execution -> rolling update.
 Status: IMPLEMENTATION ROADMAP — ready for execution, no code written yet.

--- phase-a-pre-session-truth-source-fix.md ---
 First line: "# Phase A-PRE: Session Truth Source And Durability Fix"
 Summary: Prerequisite fix before Plan+TodoList implementation. Identifies two
   critical consistency bugs: (1) session persistence happens too late in agent
   turn, not atomic enough for crash recovery; (2) GUI can prefer localStorage
   cache over backend history. Defines the rule: backend session history is
   authoritative, user input persisted before turn execution.
 Status: PRE-IMPLEMENTATION FIX — design complete, fix NOT yet implemented.
