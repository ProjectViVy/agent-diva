## 第二轮发散（2026-03-30）

**技法：** SCAMPER、形态分析、What If；**约束：** 仅 agent-diva 新 crate、OMC 全面性为内化对标。

### A. Crate 与模块切口（Substitute / Combine）

- **`diva-meta` 或 `diva-session`**：专门承载「OMC 级」生命周期（SessionStart/End、Stop、PreCompact），与 **执行循环 crate** 解耦，避免单文件上帝模块。
- **Combine**：把 **CapabilityRegistry** 和 **HookRegistry** 做成两个 registry，中间用 **事件类型** 耦合，而不是互相 `use` 深层类型。
- **Substitute**：黑板默认 `Arc<RwLock<Board>>`，测试换 `Arc<RecordingBoard>`，不碰业务代码。

### B. 形态分析小网格（3 参数）

| 维度 | 选项 A | 选项 B | 选项 C |
|------|--------|--------|--------|
| 事件投递 | 同步调用链 | 异步 channel | 混合（sync hook + async 工具） |
| Chair 输入 | 仅结构化提案 | 提案 + 原始片段 | 仅 topic id（极简 v0） |
| 冲突解析时机 | 激活 bundle 时 | 每轮 turn 前 | 工具调用前（最迟） |

**想法：** v0 可固定 **B + A + A**（结构化、同步链、激活时解析），把「每轮前重算」留给热切换能力时用。

### C. What If（故意极端）

- **What if 没有并行？** 仍保留 swarm 抽象（handoff + 顺序 Voice），协议与类型全保留，实现先顺序化 —— 降低第一条 PR 复杂度。
- **What if hook 比 turn 还多？** 规定 **事件预算**（每用户消息最多 N 个 hook 副作用），防 OMC 式脚本链失控；确定性拒绝优于默默变慢。
- **What if 用户同时挂载 50 个 bundle？** **优先级覆盖** + **激活集合硬上限** + `doctor` 报告 —— 产品上与「一个人」一致（人不能同时开 50 种人格装备，但机器可以，需护栏）。

### D. 失败模式与测试（Reverse / Black Hat）

- **Lease 泄漏**：turn 结束未交回；用 `Drop` guard 或 `scopeguard` 模式在 Rust 里硬保证。
- **双 Chair**：两个 Voice 同时认为自己是 lease 持有人；类型上让 **lease 不可 Clone**，只 `transfer`。
- **优先级环**：bundle A>B>C>A；激活阶段 **拓扑排序失败 → 拒绝加载** 并打印环。
- **Hook 死循环**：A 触发 B 触发 A；记录 **同 turn 递归深度**，超限 **硬停 + 审计**。

### E. 观测与 DX

- 每个 **EventKind** 对应 **span 名**；**capability_id**、**resolved_priority** 进 span 属性（对齐 OpenTelemetry 习惯）。
- **`SessionDiagnostics`**：导出「当前 lease 持有者、激活 bundle 列表、最后一次冲突覆盖表」— 对标 OMC 可调试性，但是纯 Rust 结构体。

### F. 为「性格」预留但不实现

- 只在 **预算与 Voice 调度** 留 `trait MoodGate` 空实现或 `()` 占位；**不改** `SteeringLease` 与 **确定性 Chair** 不变式。

### 「听不懂」条目 → v0 策略（用白话对齐）

**租约 / 安全 / 流式（问 3–5）**  
- 你的意图：**第一阶段不抠安全细案**，先 **能跑、不跑飞**。  
- **收敛**：v0 **内置最简规则**（单活跃对话路径、默认合并），**不写抢占 lease、不写对外中间流**；第四轮里偏安全的条目 **整体推迟** 到 diva 主线需要时再接。

**manifest / 冲突（问 6–9）**  
- **收敛**：v0 用 **最少字段**（名称、skills 路径或等价物、可选优先级数字）；冲突时 **简单规则**（后加载覆盖或单一优先级字段），**不进 ADR 深渊**。

**swarm 边缘（问 13–15）**  
- **收敛**：超时/空提案/错误提案/handoff 后黑板 —— **默认实现一种**（例如超时当没说话、handoff 后旧黑板只读），**不开放配置**，直到你要调。

**事件 / hooks（问 16–20）**  
- **收敛**：v0 **小事件集**（如：用户消息进入、工具调用前后、一轮结束）；**不**做插件扩展事件、不抠 Stop 子类型全家桶；**能跑后再「OMC 化」加厚**。

**记忆 / 黑板 / 压缩（问 21–24）**  
- **收敛**：**单进程 + 内存黑板 + 一条主对话历史**；压缩/双环 **后补**；审计与 trace **能记则记**，不绑复杂管道。

**测试（问 25–27）**  
- **收敛：** **怎么简单怎么来** —— 必要单元测试 + 一条 happy path 集成即可；proptest/生成式状态机 **不挡 v0**。

**安全专题（问 28–30）**  
- **收敛：** **首轮不实现**；**跟随 diva 仓库主线** 何时引入再跟。
