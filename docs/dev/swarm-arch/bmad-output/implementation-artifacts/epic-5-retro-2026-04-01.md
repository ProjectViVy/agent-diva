## 8. 关键收获（带走三条）

1. **失败路径优先建模：** 序曲中途失败时的 **计数 + 检查点 + 日志** 一次打齐，避免「成功路径文档很靓、失败路径各说各话」。
2. **文档即契约：** FR22、handoff、编排 ADR 三类读者不同，但都需要 **单点冻结**；评审时优先问「维护者会读哪一段」。
3. **热点文件要制度化解法：** `loop_turn` 的 defer 不是抱怨，是信号：**拆分或 PR 纪律** 应进入下一迭代的显式行动项。
