## 3. 故事级洞察（来自 Dev Agent Record / Review）

### 3.1 共性难点与模式

- **真相源与首屏诚实态（2.1）：** 评审指出首次 `get_cortex_state` 失败时不应呈现「已关闭」假象；已修复为 `ready` / `loadError` / 重试与 `cortex_toggled` 恢复。  
- **失败路径与安全（2.2）：** 故障注入环境变量须在 `cfg(test)` / `debug_assertions` 下生效，避免 release 误触；已纳入审查修复。  
- **分轨与性能（2.3）：** `swarm_process` 与流式 `delta`/`final` 分轨；前端 `requestAnimationFrame` 合并；多批 capped 系统消息去重、批次内单次赋值减少抖动。  
- **契约与隐私（2.4）：** `run_telemetry` 序列化失败时不应下发 `"{}"`；改为 SSE comment，避免破坏 ADR-E 与静默坏状态。

### 3.2 反复出现的评审主题

- **UI 状态必须与后端可对账**（开/关/加载失败/同步失败）。  
- **文档与实现一致**（Completion Notes 与真实 toast/行为对齐）。  
- **显式冻结「语义缺口」**（如 `lightweight` 启发式至有 wire 为止）。

### 3.3 明确记录的技术债 / 延期项

- **皮层 UI 与 gateway 过程门控可能不同步**（2.3 Completion Notes，Review Defer）：属既有架构债，依赖后续 gateway/同步故事收敛。  
- **`lightweight` 无独立后端事件**：当前启发式（无 `tool_call_*` 且非 capped）已写入 AC；待 1.8+ 显式信号再对齐 wire。

### 3.4 测试与质量

- **自动化：** 2.2（Vitest mock + Rust 单测）、2.3（组件测）、2.4（契约/编译链路）。  
- **缺口：** 2.3/2.4 均注明全链路 E2E（Tauri+网关）未加；依赖组件测 + 手动/后端贯通。  
- **工程杂项：** 2.4 为解 `BusSwarmProcessSink` 与 `MessageBus` 的 `Debug` 冲突做最小 derive 移除（与 2.3 结构一致）。
