# 飞书配置教程

> 飞书通过 WebSocket 长连接接入，无需公网 IP，适合企业内部使用。

## 平台概览

- **接入方式**: WebSocket 长连接
- **需要公网 IP**: ❌ 不需要
- **配置难度**: ⭐⭐ 简单

## 前置条件

- 一个**企业飞书账号**（个人版不支持创建自建应用）
- 飞书管理员或具有应用开发权限的账号

## 平台端申请步骤

### 第一步：登录飞书开发者后台

打开浏览器，访问 [飞书开放平台](https://open.feishu.cn/)，使用你的企业飞书账号登录。

### 第二步：创建企业自建应用

1. 点击页面顶部的 **「开发者后台」**
2. 点击 **「创建企业自建应用」** 按钮
3. 填写应用信息：
   - **应用名称**: 如 `Agent Diva Bot`
   - **应用描述**: 如 `AI 智能助手`
   - **应用图标**: 上传一个合适的图标（可选）
4. 点击 **「创建」**

### 第三步：获取应用凭证

1. 创建完成后，进入应用详情页
2. 在左侧菜单中找到 **「凭证与基础信息」**
3. 记录以下信息：
   - **App ID**: 格式如 `cli_a5xxxxxxxxxxxxx`
   - **App Secret**: 点击"显示"后复制

⚠️ **重要**: App Secret 是敏感信息，不要泄露。

### 第四步：添加机器人能力

1. 在左侧菜单中找到 **「应用能力」→「添加应用能力」**
2. 找到 **「机器人」** 能力，点击 **「添加」**

### 第五步：配置应用权限

在左侧菜单中找到 **「权限管理」**，搜索并开启以下权限：

| 权限标识 | 权限名称 | 用途 |
|---------|---------|------|
| `im:message` | 获取与发送消息 | 接收和发送聊天消息 |
| `im:message.create_v1` | 以应用身份发消息 | 机器人主动发送消息 |
| `im:resource` | 获取消息中的资源文件 | 接收图片、语音等媒体 |

### 第六步：配置事件订阅（关键步骤！）

1. 在左侧菜单中找到 **「事件与回调」**
2. ⚠️ **在「事件配置方式」中，选择「使用长连接接收事件」**
   - 这是关键步骤！选择长连接模式后，无需公网 IP
3. 在「添加事件」中搜索并添加：
   - **im.message.receive_v1** — 接收消息

### 第七步：创建版本并发布

1. 在左侧菜单中找到 **「版本管理与发布」**
2. 点击 **「创建版本」**
3. 填写版本号（如 `1.0.0`）和更新说明
4. 设置 **可用范围**: 选择允许使用该应用的部门或人员
5. 点击 **「保存」**，然后 **「申请发布」**
6. 等待管理员审批

## Agent Diva 配置

### 方式一：GUI 配置（推荐）

1. 打开通道配置页面
2. 点击"添加通道"选择飞书
3. 填写 App ID 和 App Secret
4. 点击"下一步"测试连接
5. 测试通过后完成配置

### 方式二：手动编辑配置文件

编辑 `~/.agent-diva/config.json`：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "app_id": "cli_a5xxxxxxxxxxxxx",
      "app_secret": "你的 App Secret",
      "verification_token": "可选的验证 Token"
    }
  }
}
```

## 验证与测试

1. 启动 Agent Diva，观察日志输出
2. 打开飞书，在搜索栏中搜索应用名称
3. 点击进入对话，发送一条消息
4. 观察日志和机器人回复

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| 搜索不到机器人 | 确认应用已发布且你在可用范围内 |
| 消息收不到 | 检查是否选择了「长连接模式」（不是 Webhook） |
| 权限不足 | 确认所有必要权限已开通，且应用已重新发布 |
| 日志显示连接失败 | 检查 App ID 和 App Secret 是否正确 |
