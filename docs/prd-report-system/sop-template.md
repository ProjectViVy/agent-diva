# SOP Template — Report System (v1)

> **来源**: `docs/prd-report-system/prd.md` §4.2 FR-6
> **版本**: v1 (2026-06-12)
> **用途**: 从报表内容固化为 SOP 的标准模板

---

## 模板结构

```markdown
# {SOP 标题}

> 从 {源报表类型} {源报表日期} 固化的 SOP
> 生成时间: {ISO8601 timestamp}
> 源文件: {源报表路径}

## 目的

{从报表中提取的目的描述。回答"为什么需要这个 SOP"和"它解决什么问题"。通常 1-2 段。}

## 前置条件

- {依赖的报表}: `report-system://{period}/{date_or_week_or_month}`
- {依赖的工具/技能}: `skill://{skill-name}` (如有)
- {环境要求}: {操作系统/软件版本/网络要求}

## 步骤

1. **{步骤标题 1}**
   {详细描述}
   ```bash
   # 可选命令示例
   ```
   
2. **{步骤标题 2}**
   {详细描述}
   - {子步骤 2.1}
   - {子步骤 2.2}

3. **{步骤标题 3}**
   {详细描述}

## 注意事项

- ⚠️ {重要警告}: {说明}
- 💡 {提示}: {说明}
- 🚫 {禁止}: {不要做什么}

## 预期结果

{执行完本 SOP 后预期达到的状态}

## 回滚

{如果中间步骤失败, 如何回滚}

## 参考链接

- 源报表: `{workspace}/reports/{path}`
- 相关 SOP: `{workspace}/sops/{related-sop}.md`
- 相关 Skill: `{workspace}/skills/{related-skill}/SKILL.md`
- 相关 ADR: `docs/adr/{adr-id}.md`
- 外部参考: {URL}

## 元数据

- **作者**: {User 名称}
- **创建时间**: {ISO8601 timestamp}
- **最后更新**: {ISO8601 timestamp}
- **状态**: draft | review | stable | deprecated
- **标签**: {tag1, tag2, tag3}
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 标题 | ✅ | 简短动词开头 (e.g. "配置 X", "部署 Y") |
| 目的 | ✅ | 1-2 段说明, 不超过 200 字 |
| 前置条件 | ✅ | 列出所有依赖, 包括报表/Skill/环境 |
| 步骤 | ✅ | 3-7 步, 每步一个原子动作 |
| 注意事项 | 🟡 | 警告/提示/禁止 三类 |
| 预期结果 | ✅ | 验证 SOP 成功的客观标准 |
| 回滚 | 🟡 | 涉及系统变更时必填 |
| 参考链接 | 🟡 | 至少包含源报表路径 |
| 元数据 | ✅ | 作者/时间/状态/标签 |

---

## 模板生成流程

1. **LLM 提取**: 从源报表中提取目的/步骤/注意事项
2. **用户编辑**: GUI 弹出编辑器, 用户可微调各字段
3. **用户审阅**: 用户在保存前 review 完整内容
4. **写入文件**: 保存到 `{workspace}/sops/{report-id}-{timestamp}.md`
5. **冲突检测**: 若同名文件存在, 弹冲突对话框 (覆盖/重命名/取消)

---

## 模板与 v1.1 schema 的关系

本 SOP 模板**不强制**使用 §4.4 的 report frontmatter schema (SOP 不是报表)。
但 SOP 内部的"源报表"链接使用 `report-system://` URI scheme, 与 ADR-008 的写入路径保持概念一致。
